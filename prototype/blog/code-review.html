Code review bez dramatů

18. listopadu 2025 | git workflow

Code review nemusí být boj. Malé pull requesty, jasný popis změny a jedno pravidlo: review max. do 48 hodin. Kdo nechá PR ležet týden, brzdí celý tým.

Komentáře pište jako návrhy, ne jako rozsudky. „Co kdybychom…" funguje líp než „tohle je špatně". Výsledek? Rychlejší merge, lepší kód a tým, který se nebojí pushovat.
