Project management v CLI — jde to

7. února 2026 | workflow

Stačí adresář ~/Projects se strukturou zákazka/projekt a prj.py. Každý projekt má svůj PROJECT.yaml s backlogem, next steps a changelogem.

~/Projects/
  redhat/            ← zákazka
    ├── golang_http/ ← projekt
    └── docker_man/  ← projekt

Děláš více projektů současně? Rychle zapiš nápad, přepni se a vrať se bez ztráty kontextu. Žádné zapomenuté nápady, žádné ztracené odpoledne.

prj - Správce projektů

Použití: prj <příkaz> [argumenty]

Struktura: ~/Projects/<zákazka>/<projekt>/PROJECT.yaml

Příkazy:
  list                              Vypíše všechny projekty
  stats                             Zobrazí stav všech projektů
  show <projekt>                    Zobrazí detail projektu
  create                            Vytvoří nový projekt (interaktivně)
  rename <starý> <nový>             Přejmenuje/přesune projekt
  check <projekt>                   Ověří strukturu PROJECT.yaml
  migrate [projekt]                 Migruje README.md → PROJECT.yaml
  readme <projekt> | --all          Regeneruje README.md z PROJECT.yaml

  task <projekt>                    Zobrazí backlog
  task add <projekt> "úkol"         Přidá úkol do backlogu
  task next <projekt> <číslo>       Přesune z backlogu do Next steps
  task done <projekt> <číslo>       Dokončí úkol → changelog

  log <projekt>                     Zobrazí changelog
  log add <projekt> "zpráva"        Přidá záznam do changelogu
